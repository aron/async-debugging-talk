Debugging JavaScript

Types of Error

  • Syntax Errors — typos and mistakes in the code. 
    
					// Missing parentheses, braces etc.
					console.log([1, 2, 3, 4];
  • Runtime Errors — trying to use non-existent properties. 
    
					// Calling an undefined method.
					var obj = {};
					obj.method();

Console

  • console.log() — prints out to the console. 
    
					console.log("An object: %o and an array: %o", {a: 1}, [1, 2, 3]);
  • console.dir() — inspects the properties on an object. 
    
					console.dir({a: 1, b: 2, c: "skips"});
  • console.dirxml() — inspects a DOM node. 
    
					console.dirxml(document.querySelectorAll(".wotsits"));
  • console.assert() — validates a statement as true. 
    
					console.assert(quavers === "wholesome", "Oh no, expression isn't true");

Command Line

  • dir(), dirxml() — identical to their console counterparts. 
    
					// Save some typing.
					dir("Yada yada");
					dirxml(document.getElementsByTagName("script"));
  • $n, $(), $$(), $x() — functions to query the DOM. 
    
					$0; //=> The currently selected DOM node in the inspector.
					$1; //=> The previously selected DOM node in the inspector.
					$("mccoys"); //=> Alias of document.getElementById().
					$$("div .frazzles"); //=> Alias of document.querySelectorAll().
					$x("/noone/likes/xpath/anyway"); //=> An xpath query.
  • inspect() — jumps to the relevant info pane. 
    
					inspect($$('body')[0]); //=> Jump to the body in the Elements pane.

Debugger

  • Step In/Out — step in and out of the current function
  • Step Over — skip over the next function
  • Continue — continue on to the next breakpoint
  • Watch — track changes to a variable
